Buttons with Press Feedback

scale(0.97) on :active

Buttons must feel responsive. A subtle scale on press gives instant feedback, making the UI feel like it's truly listening.

Origin-Aware Popovers

transform-origin from trigger

Popovers should scale in from their trigger, not from center. The default transform-origin: center is wrong for almost every popover.

This popover scales from the trigger above, not from its center. Notice how it feels more connected.

Hold-to-Delete Pattern

clip-path reveal

Press should be slow when deliberate (2s linear), but release should always be snappy (200ms ease-out). Hold the button to see the danger overlay reveal.

clip-path: inset(0 100% 0 0) → inset(0 0 0 0)

Tab Indicator

Smooth sliding indicator

The indicator slides behind tabs rather than using color transitions on individual tabs. This creates a more polished, cohesive feel.

transition: transform 250ms cubic-bezier(0.23, 1, 0.32, 1)

Tooltips with Instant Subsequent Hovers

Skip delay after first tooltip

First tooltip has a 400ms delay. But once one is open, hovering over adjacent tooltips opens them instantly. This feels faster without defeating the purpose of the initial delay.

Modal (Centered Transform Origin)

Exception: Modals stay centered

Unlike popovers, modals should keep transform-origin: center because they're not anchored to a specific trigger — they appear centered in the viewport.

transform-origin: center center (unlike popovers)

Toast Notifications

Sonner-style toasts

Toasts enter from bottom with translateY(100%). CSS transitions (not keyframes) allow smooth interruption when adding multiple toasts rapidly.

translateY(100%) → translateY(0), 400ms ease-out

Hover Cards

Touch device protection

Hover animations are gated behind @media (hover: hover) and (pointer: fine). Touch devices trigger hover on tap, causing false positives without this.
